# Reference Surfaces and Editable Boundaries

This repository separates canonical editable sources from read-only reverse-engineering references.

| Path | Role | Edit policy |
|---|---|---|
| `app/apktool/` | Canonical editable/buildable Apktool tree | Editable |
| `reference/jadx-raw/` | Raw JADX reference export | Read-only |
| `reference/jadx-aliased/` | Aliased JADX reference export | Read-only |
| `reference/firstparty-jadx/` | Focused first-party reference output | Read-only |
| `reference/vendor-jadx/` | Focused vendor/TW reference output | Read-only |
| `mappings/jadx/` | JADX alias cache (`*.jobf`) | Do not hand-edit |
| `mappings/manual-enigma/` | Manual evidence-backed mappings | Editable |
| `docs/maps/` | Generated component/file maps | Generated docs |
| `docs/reports/` | Generated readability/vendor/release reports | Generated docs |

JADX packages such as `com.p060tw.music` / `com.p073tw.music` are reference artefacts, not the runtime package. Runtime package identity remains `com.tw.music`.

`reference/jadx-mt/` is optional and may be absent unless generated by the MT import workflow.
